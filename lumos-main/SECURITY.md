## 报告漏洞

如果你发现了安全漏洞，请**不要**在公开的 Issue 中报告。

### 报告方式

1. 通过 [GitHub Security Advisories](https://github.com/7836246/lumos/security/advisories/new) 提交私密报告
2. 或发送邮件至项目维护者

### 报告内容

请在报告中尽量包含以下信息：

- 漏洞类型（如 XSS、SQL 注入、信息泄露等）
- 受影响的文件或组件路径
- 复现步骤
- 概念验证代码（如适用）
- 潜在影响评估

### 响应时间

- **确认收到**：48 小时内
- **初步评估**：7 天内
- **修复发布**：视严重程度而定，严重漏洞将在 30 天内修复

## 安全最佳实践

本项目高度重视用户隐私与数据安全：

- 📱 **端侧 OCR**：合同文字识别在用户设备本地完成，原始图片不上传
- 🔒 **数据脱敏**：敏感个人信息（身份证号、银行卡号等）在传输前自动脱除
- 🚫 **不存储原文**：服务器不持久化完整合同原文
- 🔐 **传输加密**：所有通信强制使用 HTTPS/TLS
