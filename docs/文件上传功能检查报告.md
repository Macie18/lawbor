## ⚠️ 发现的问题

### 🔴 严重问题

#### 1. 前端接受的文件格式与后端不匹配

**问题描述**:
- 合同审查页面接受 `.pdf,.doc,.docx,.jpg,.png`
- 但后端 fileParser.ts 只支持 `.pdf` 和 `.docx`
- 用户上传 `.doc` 或图片会收到错误提示,体验不佳

**影响范围**: ContractReview.tsx

**建议修复**:
```tsx
// 修改前
accept=".pdf,.doc,.docx,.jpg,.png"

// 修改后
accept=".pdf,.docx"
```

---

#### 2. PDF Worker 加载方式不一致

**问题描述**:
- `fileParser.ts` 使用 CDN 加载 worker:
  ```ts
  pdfjsLib.GlobalWorkerOptions.workerSrc = 
    `https://unpkg.com/pdfjs-dist@${pdfjsVersion}/build/pdf.worker.min.${workerExt}`;
  ```
- `pdfParser.ts` 使用本地 worker:
  ```ts
  pdfjsLib.GlobalWorkerOptions.workerSrc = new URL(
    'pdfjs-dist/build/pdf.worker.min.mjs',
    import.meta.url
  ).toString();
  ```

**风险**:
- CDN 方式可能在用户网络受限时失败
- 中国大陆用户访问 unpkg CDN 可能超时
- 生产环境应优先使用本地 worker

**建议**: 统一使用 pdfParser.ts 的本地 worker 方式

---
