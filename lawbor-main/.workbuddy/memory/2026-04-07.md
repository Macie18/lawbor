## 第三次修复（22:13）

### 合同审查 Dify 返回格式错误
- **根因**：`callDifyAPIStream` 里 `buffer.split('\n')` 拆出的行带有 CRLF 中的 `\r` 尾巴，导致 `trimmed.startsWith('data: ')` 失败，`fullContent` 始终为空，`parseDifyResponse` 因文本过短返回 null
- **修复**：每行做 `.replace(/\r$/, '').trim()` 去 CRLF；用 `data.event === 'message'` 精确匹配而非 `data.answer` 非空判断；检测 `message_end` 和 `error` 结束事件
- **Dify 配置本身没问题**：直接 curl 测试，API Key 正常，返回格式正确

### 模拟面试消息无响应（根因）
- **根因**：laboris-api/src/index.ts 读的是 `process.env.VITE_DEEPSEEK_API_KEY`，但 laboris-api/.env 里配置的是 `DEEPSEEK_API_KEY`（Node 服务不该用 VITE_ 前缀），导致 key 为空，所有 chat 请求返回 503
- **修复**：改为 `process.env.DEEPSEEK_API_KEY || process.env.VITE_DEEPSEEK_API_KEY`，重新 build 并重启服务
- **验证**：curl localhost:4000/v1/chat 成功返回 AI 回复

### 语音 network 错误
- **根因**：Chrome Web Speech API 偶发 network 错误（访问 Google 语音服务不稳定），不是真正的应用错误
- **修复**：`useSpeechDictation.ts` 把 `network` 加入静默错误列表；`index.tsx` 的 onError 也过滤 network/no-speech

### 服务状态

- Vite dev server 运行在 localhost:3000

## 第四次修复（22:21）

### 合同审查 Dify 再次报"返回文本过短"
- **根因确认**：不是 CRLF 问题（代理 curl 测试和直连均正常），而是浏览器 fetch 读取 Vite 代理的 SSE 流存在透传不稳定问题（可能是 HMR 缓存或浏览器 fetch+proxy 的兼容性）

- **降级保留**：blocking 失败时自动降级到 streaming 模式
- **同时**：给 Vite proxy 加了 `configure` 强制 `content-encoding: identity` 禁用压缩缓冲
- **调试日志**：`callDifyAPIStream` 里保留了详细 chunk 日志便于后续排查

## 第六次修复（22:36）

### 模拟面试说话后 AI 无响应（语音识别层致命 bug）
- **根因**：多层 bug 叠加
  1. `cleanup()` 用 `abort()` 会触发 `onend`，导致 `cleanup` 和 `onend` 交叉调用
  2. `network` 错误触发 `onerror` → `cleanup()` → `abort()` → `onend`，但 `onend` 里没有调用 `commitAndStop()`，导致 `onCommit` 从未被调用，识别结果完全丢失
  3. `commitAndStop` 用 `liveLine` state 变量（闭包延迟问题），可能拿到空值
- **修复**：
  1. `cleanup()` 改用 `stop()`（不触发 `onend`）代替 `abort()`
  2. 加 `liveLineRef` 同步保存当前识别文本
  3. `onerror` 里 `network` 错误时：用 `liveLineRef` 立即取出已识别文本，`setTimeout(() => onCommit?.(lastText), 0)` 异步提交
  4. `commitAndStop` 改用 `liveLineRef.current` 代替 `liveLine` state
- **影响路径**：`src/hooks/useSpeechDictation.ts`

### Dify 模型未配置（blocking 失败降级也炸）
- **错误**：`Model gpt-4 credentials is not initialized`
- **根因**：Dify Agent 配置的模型是 GPT-4，但 Dify 平台没有配置 OpenAI GPT-4 凭证
- **说明**：需要去 Dify 控制台把 Agent 的模型换成 DeepSeek 或已配置好的模型
