## 用户需求

税务计算器的城市选择列表需要补充完整34个省级行政区（23省5自治区4直辖市2特别行政区）。

## 产品概述

在现有税务计算器基础上，扩展城市选择器显示完整的中国省级行政区列表，同时保持后端计算逻辑不变，通过数据映射复用现有7个基准城市的社保公积金数据。

## 核心功能

- 前端显示34个省级行政区完整列表
- 建立省份到基准城市的数据映射关系
- 补充所有省份的中英文翻译
- 保持现有计算逻辑和数据结构不变

## 技术栈

- 前端框架：React + TypeScript
- 数据存储：复用现有 SOCIAL_RATES 对象
- 国际化：TranslationContext

## 实现方案

### 核心策略

创建省份映射表，将34个省级行政区映射到现有7个基准城市，前端遍历映射表生成完整列表，后端通过映射获取社保数据。

### 映射规则

**直辖市（4个）→ 自身或相近一线城市**：

- 北京、上海、天津、重庆

**省会及重点城市（27个）→ 经济水平相近城市**：

- 广州、深圳、杭州、成都等使用自身数据
- 其他省会城市根据经济水平映射到相近基准城市

**特别行政区（2个）→ 深圳/广州**：

- 香港、澳门

**自治区（5个）→ 区域中心城市**：

- 内蒙古→北京、广西→广州、西藏→成都、宁夏→成都、新疆→成都

### 数据结构

```
PROVINCE_MAPPING = {
  'beijing': { name: '北京', baseCity: 'beijing' },
```

## 关键代码结构

### PROVINCE_MAPPING 接口定义

```typescript
interface ProvinceMapping {
  [key: string]: {
    name: string;      // 省份key
    baseCity: string; // 映射的基准城市key
  };
}

export const PROVINCE_MAPPING: ProvinceMapping = {
  // 4个直辖市
  'beijing': { name: '北京', baseCity: 'beijing' },
  'tianjin': { name: '天津', baseCity: 'beijing' },
  'shanghai': { name: '上海', baseCity: 'shanghai' },
  'chongqing': { name: '重庆', baseCity: 'chengdu' },
  
  // 23个省
  'hebei': { name: '河北', baseCity: 'beijing' },
  'shanxi': { name: '山西', baseCity: 'beijing' },
  // ... 其他省份
  
  // 5个自治区
  'innermongolia': { name: '内蒙古', baseCity: 'beijing' },

  // 2个特别行政区

```

## 实现注意事项

1. **数据映射合理性**：根据地理位置、经济水平、社保政策相似性进行映射
2. **前端兼容性**：城市选择器改为遍历 PROVINCE_MAPPING 而非 SOCIAL_RATES
3. **翻译完整性**：34个省份都需要中英文翻译键
4. **向后兼容**：保留 SOCIAL_RATES 原有结构，仅新增映射层
5. **用户体验**：省份列表按地域分组或拼音排序，便于查找
